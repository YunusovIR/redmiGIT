Это поможет вам понять как работатет GITHUB

1. как создать репозиторий
/для этого надо в корневом коталоге, где будут хронится ваши файлы создать папку "my gitHub project".
2.переходим в папку которую мы создали и нажатием клавиш Ctrl+Shift+F10 вызываем окно команд.
/в окне команд прописываем git init // каждая команда начинается с оператора "git"
3. после ввода команды инициализации внешей папке появится скрытая диектория [ .git ] 
в папке .git хранится вся информация по вашему проекту. Самыми важными элементами являются objects, refs, HEAD, index. 
4.в нашей папке "my gitHub project" создаем текстовый документ и называем его "CODE"
// в коммандной строке вводим "git status"
это код нам выдаст следущую строку 

C:\Users\yunusov\Desktop\git\git testing>git status
On branch master

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        code.txt

nothing added to commit but untracked files present (use "git add" to track)

в этих строках нам говорится что есть неотслеживаемые файлы, и предлогается выполнит комманду
git add <file> // внашем случае это будет выглядеть git add code.txt
далее мы должны принять изминения вводим команду git commit -m "coments"
сново вводим git status и видим что все изминения приняты.

любое изминение в файле должно быть commit "совершит".
 
git init
git status
git add <File.*> //если файлов больше двух можно вести команду "git add ."
git commit -m "cpments" // всегда любой commit коментировать.

как создать ветвь в репозитории
1. введите коменду git branch name
2. проверка указания ветки git log --oneline --decorate // эта команда нам говорит где мы находимся. (HEAD -> master, name)
3. как переключится с ветки master на ветку name // git checkout name
4. проверяем что мы в ветви name // git log --oneline --decorate // (HEAD -> name, master)

как удалить ветвь в репозитории
1. git breanch -d name // -d delet
2. проверяем git log --oneline -- decorate // вы удаляли ветвь name находясь в ней проверка вам покажет что ветвь еще существует
перейдите в ветвь master и повторите проверку.

как залить файлы в GitHub

1. перейдите на сайт github.com
2. пройдите регистрацию
3. создайте новый репозиторий
4. выберите //
…or push an existing repository from the command line
введите в командной строке
git remote add origin https://github.com/ваш профиль/название репозитория
git push origin master //если вы создали ветвь и хотите с нее слитьфайлы вводите git push origin ваша ветвь
